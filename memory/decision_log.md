## 2026-06-07 プロジェクト方針の確定（mouth_shooting）

leaflet-notifier 用だった設定（CLAUDE.md / .claude/rules）を、マウス操作ブラウザ
シューティングゲーム用に書き換えた。確定事項は以下。

### プロダクト
- **マウス操作のブラウザ・シューティング**。自機はカーソルに追従。
- 左クリック=ショット（押しっぱなしで連射、最寄り敵へオートエイム）。
- 右クリック長押し=ボム（自機中心から衝撃波の輪、半径∝溜め時間、溜め中はショット不可・移動減速）。
- **コア経済**: ショット＝コンボを積む準備役（微小な稼ぎ）→ コンボがボム溜め時間を短縮 →
  ボム＝主得点源（範囲一掃）。ステージ進行で敵数・突進速度・弾数が増え、ショットとボムを編むことを強制。

### 確定した設計判断
| 項目 | 決定 | 理由 |
|------|------|------|
| 技術 | Canvas 2D + 素の ES Modules JS、依存ゼロ・ビルド不要 | $0 公開・保守簡素 |
| 構造 | ステージ制（各ステージ末にボス）、進行で難易度上昇 | ユーザー選択 |
| ビジュアル | ネオン幾何・コード描画（画像ファイル不使用） | 依存ゼロ・即ロード |
| 音 | WebAudio で生成。コンボで音楽レイヤーが増える動的音楽＋効果音 | ユーザー選択（音でも報酬を体感） |
| コンボ報酬 | (A) ボム溜め時間短縮 + (B) 動的音楽レイヤー増加 | ユーザー選択 |
| 公開 | public リポジトリ + GitHub Pages（無料で遊べる URL） | $0・誰でもプレイ可 |

### 操作の設計判断（ユーザー未指定 → デフォルト採用）
- 自機=カーソルのため手動エイム不可 → **ショットは最寄りの敵へオートエイム**（vampire-survivors系）。
  敵が四方から突進する設計と整合。違和感あれば「上方向固定」等へ変更可。

## 2026-06-08 ステージ制 → エンドレスへ変更（ユーザー要望）

初版はステージ制（各ステージ末にボス）だったが、ユーザー要望で**エンドレス**に変更。

- **エンドレス進行**: `src/stage.js` を時間経過ベースの難易度ディレクターに再構成。
  `difficultyAt(elapsedSec)`（純粋関数）で、出現間隔↓・突進速度↑・射撃割合↑・1体の弾数↑ を連続的に難化。
  ステージ区切り（STAGE CLEAR 待ち）と `allclear` 終了状態は廃止。終了は GAME OVER のみ。
- **ボス**: 進行ゲートではなく、一定間隔（既定40秒）で出現する難化イベントに変更。
- **ボムで敵弾も消去**: 衝撃波が敵弾も巻き込む（防御の意味を付与）。
- **ボムのスコア累積**: 1回のボム内で撃破が進むほど1体の得点が累積（撃破順 ×1, ×2, ×3 …）。
  大掃除が爆発的に稼げる。`bigSweep` 以上で「BOMB xN!」表示。`killScore` を 22→14 に調整。
- HUD は STAGE 表示をやめ、TIME（mm:ss）と Lv. 表示に変更。
- テストは `tests/stage.test.mjs` を `difficultyAt` の単調性テストに差し替え（全24件 pass）。
