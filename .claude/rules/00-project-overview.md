# Project Overview

## Project name

mouth_shooting（マウス操作のブラウザ・シューティングゲーム）

## Goal

マウスだけで遊べるブラウザ・シューティングを作る。
自機はカーソルに追従し、左クリックでショット（押しっぱなしで連射）、右クリック長押しでボム（衝撃波）を放つ。
コンボを積むほどボムが早く溜まり、音楽のレイヤーが増えて高揚する——リスクとリターンの綱引きで中毒性を出す。

依存ゼロ・ビルド不要で、**GitHub Pages に置くだけで誰でも遊べる**ことを最優先する。

## コア・ゲームループ

```
ショット（最寄り敵へオートエイム）で敵を倒す
  → ボムエネルギー(燃料)が溜まる ＋ コンボが積まれる ＋ 微小スコア
        ↓
コンボがボムの溜め速度・エネルギー回復を強化 + FM音楽レイヤーを追加
        ↓
ボム（衝撃波で範囲一掃。敵弾も消す）で大きく稼ぐ ← 主得点源。エネルギーを消費
  ・1回のボムで巻き込んだ撃破数が多いほど、1体あたりの得点が累積的に上がる
        ↓
エンドレス: 時間経過で敵数・突進速度・敵弾・敵タイプが増加（ボスは一定間隔で出現）
  → ボム連打＝燃料切れ＆その間ショットできず補充不可 / ショットだけ＝稼げない
  → ショット↔ボムの往復を強制
```

## 操作

- **自機移動**: マウスカーソルに追従（軽いスムージング）。ボム溜め中は移動が遅くなる。
- **ショット（左クリック、押しっぱなしで連射）**: 最寄りの敵へオートエイム。撃破で微小スコア＋コンボ+1＋**ボムエネルギー回復**。
- **ボム（右クリック長押しで溜め→離す）**: 自機中心から光の輪が拡大。半径 ∝ 溜め量で、**ボムエネルギーを消費**（燃料が無いと撃てない）。溜め中はショット不可。範囲内の敵と敵弾を一掃し高スコア＋コンボ大増加。
- **敵タイプ**: drifter（直進）/ weaver（蛇行）/ charger（突進）/ turret（砲台・速い狙い弾）/ spinner（旋回・全方位弾）/ striker（螺旋弾）。時間で解禁。
- **失敗条件**: 敵接触 or 被弾で life 減。life 0 で GAME OVER。ハイスコアは localStorage に保存。

## 技術スタック / 実行形態

- 言語: JavaScript（ES Modules）。ビルド・依存パッケージなし。
- 描画: Canvas 2D。ネオン幾何（黒背景＋発光）をコードで描く。画像ファイル不使用。
- 音: WebAudio API でコード生成（効果音 + コンボ連動の多層動的音楽）。音源ファイル不使用。
- 実行: ブラウザで `index.html` を開くだけ。公開は GitHub Pages（静的配信）。
- 保存: ハイスコア等は `localStorage` のみ。サーバ・DB・秘密情報を持たない。

## 非機能要件・方針

- **低コスト**: $0。サーバ・有料サービスを使わない。
- **依存ゼロ・ビルド不要**: フレームワーク／バンドラを導入しない（YAGNI）。個人ゲームに過剰な汎用化をしない。
- **公開時の posture**: `README.md` に操作・遊び方・免責、`LICENSE`（MIT）を同梱。秘密情報を含めない。
- **手触り優先**: 数値（速度・チャージ係数・難易度カーブ）は `src/config.js` に集約し、一箇所で調整できるようにする。
